# Analysis Framework

## 1. Define

先把自然语言问题翻译成可计算问题：

- 对象：用户、订单、事件、页面、交易、任务。
- 指标：数量、金额、转化率、时长、留存、错误率。
- 范围：时间、地区、渠道、版本、业务线。
- 粒度：天、周、月、用户、会话、订单。
- 对比：环比、同比、A/B、前后版本、不同分组。

## 2. Inspect

最小检查集：

- 行数、列数、文件大小或表大小。
- 字段类型和样例。
- 每列缺失率。
- 主键或候选键重复率。
- 时间字段的最小值、最大值和空洞。
- 数值字段的最小值、最大值、均值、中位数、分位数。

## 3. Clean

只做必要清洗，并记录规则：

- 明确空值如何处理。
- 明确重复数据是否去重。
- 明确异常值是剔除、截尾还是保留。
- 明确时区、币种、单位和大小写归一。

不要静默改口径。口径变化必须写进结论。

## 4. Analyze

常见分析套路：

- 总览：总量、均值、中位数、分位数、趋势。
- 分组：按渠道、地区、版本、客户类型、时间分桶。
- 漏斗：每一步人数、转化率、流失率。
- 留存：cohort、首日/7日/30日。
- 异常：与历史基线或同组对比，找突增、突降、离群。
- 相关：只说相关，不直接说因果，除非实验设计支持。

## 5. Validate

结论前至少做一轮反查：

- 数字是否能用另一条查询复现。
- 分母是否正确。
- 是否被缺失值、重复值或极端值主导。
- 是否有辛普森悖论：总体趋势和分组趋势相反。
- 是否把采样数据说成全量数据。

## 6. Present

输出结构：

1. 一句话结论。
2. 关键证据，列出指标和范围。
3. 图表或表格，展示趋势、对比或结构。
4. 风险和限制。
5. 下一步建议。
